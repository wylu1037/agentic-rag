# 场景边界定义（Week 0）

## 目标场景

- 场景名称：Agentic RAG 项目内部知识问答助手
- 目标用户：研发、产品、实施同学
- 核心任务：针对项目文档进行问答，并返回可追溯引用

## In Scope

1. 回答“项目实现路线、技术选型、阶段计划、里程碑”相关问题。
2. 输出引用来源（至少到文件级，推荐到章节级）。
3. 支持中文问答。
4. 能处理事实性问题与流程性问题。

## Out of Scope（第 0 周不做）

1. 跨系统执行型 Agent（如自动发版、自动工单处理）。
2. 多模态问答（图片/音频理解）。
3. 强实时外部数据问答（新闻、股价、实时搜索）。
4. 多租户权限体系与企业级审计。

## 数据边界

1. 当前知识源：仓库内文档（优先 README 与 docs 目录）。
2. 第 0 周不引入外部网页抓取与第三方知识库。
3. 每条评测题必须可映射到已存在文档。

## 风险与缓解

1. 风险：文档内容稀疏导致评测覆盖不足。  
   缓解：先扩充 `docs` 再扩展评测题库。
2. 风险：问题集偏“考试题”，不够真实。  
   缓解：后续从真实对话日志补充样本并替换 seed。
